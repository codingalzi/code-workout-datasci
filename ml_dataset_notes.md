# 머신러닝 분류/회귀 데이터셋 논의 메모

## 현재 방향

- 회귀 노트북은 `ml_regression.ipynb`를 유지하고, 파머 펭귄 데이터셋으로 `body_mass_g`를 예측한다.
- 분류 노트북은 펭귄 데이터셋 대신 다른 데이터셋을 사용하는 방향이 좋다.
- 이유: 펭귄 종 분류는 정확도가 100% 또는 거의 100%에 가까워, 혼동 행렬과 정밀도/재현율의 필요성을 설명하기 어렵다.

## 펭귄 데이터셋

### 장점

- 특성이 직관적이다.
- `bill_length_mm`, `bill_depth_mm`, `flipper_length_mm`, `body_mass_g` 같은 신체 측정값이 이해하기 쉽다.
- 회귀 예제에서 몸무게 예측 문제로 쓰기 좋다.

### 단점

- `species` 분류는 너무 쉽다.
- 정확도가 거의 1.0에 가까워 분류 평가 지표를 설명하기 어렵다.

### 사용 방향

- 분류보다는 회귀 노트북에 적합하다.
- `ml_regression.ipynb`에는 수치형 특성만 사용한 모델과, `island`, `sex` 같은 범주형 특성을 원-핫 인코딩해 추가한 모델을 비교하는 흐름을 넣는다.

## Iris 데이터셋

### 장점

- 분류 모델 첫 소개용으로 적절하다.
- scikit-learn 내장 데이터셋이다.
- 결측치가 없고 모든 특성이 수치형이다.
- 특성이 4개라 단순하다.
- `sepal length`, `sepal width`, `petal length`, `petal width`가 비교적 직관적이다.
- 다중 분류와 혼동 행렬 입문에 좋다.

### 단점

- 데이터가 쉬워 정확도가 높게 나올 수 있다.
- 특히 `setosa`는 다른 품종과 잘 분리된다.

### 사용 방향

- 분류 개념을 처음 소개하는 장에는 적절하다.
- 처음에는 2개 특성만 사용하고, 나중에 전체 4개 특성을 사용해 성능을 비교하는 구성이 좋다.

## Wine 데이터셋

### 장점

- scikit-learn 내장 데이터셋이다.
- 다중 분류 데이터셋이다.
- 모든 특성이 수치형이다.
- 특성 스케일이 달라 표준화 설명에 좋다.

### 단점

- 특성 이름이 초보자에게 직관적이지 않다.
- 모델 설정에 따라 성능이 높게 나올 수 있다.

### 사용 방향

- 펭귄보다 분류 예제로는 낫지만, 첫 분류 개념 소개에는 Iris가 더 쉽다.
- 분류 흐름 학습용으로는 괜찮고, 평가 지표의 필요성을 강하게 보여주려면 일부 특성만 사용하는 편이 좋다.

## Titanic 데이터셋

### 장점

- 현실적인 데이터 탐색과 전처리 연습에 좋다.
- 결측치, 범주형 변수, 그룹별 생존율 분석에 적합하다.

### 단점

- `Age`, `Cabin`, `Embarked` 결측치 처리와 `Name`, `Ticket`, `Cabin` 같은 특성 선택 문제가 먼저 부각된다.

### 사용 방향

- 이미 EDA 챕터의 연습문제로 사용 중이다.
- 분류 첫 장의 본 데이터셋으로 다시 쓰는 것은 피하는 편이 좋다.

## Breast Cancer 데이터셋

### 장점

- scikit-learn 내장 데이터셋이다.
- 이진 분류라 정확도, 혼동 행렬, 정밀도, 재현율 설명에 좋다.
- 입력 특성은 모두 수치형이다.
- 주요 특성 몇 개만 사용하면 적당히 오분류가 생겨 평가 지표를 설명하기 좋다.

### 단점

- 유방암 데이터셋이라 성별 특정 질병처럼 보일 수 있다.
- 특성이 30개이고 이름이 초보자에게 덜 직관적이다.
- 의료 데이터이므로 실제 진단을 대신하지 않는다는 안내가 필요하다.

### 현재 작성된 노트북

- `ml_cancer.ipynb`를 생성했다.
- 선택한 4개 특성:
  - `mean radius`
  - `mean texture`
  - `mean concavity`
  - `mean symmetry`
- 이 4개는 중요도가 가장 높은 특성이라기보다, 설명을 위해 서로 성격이 다른 특성을 고른 것이다.
- 문구는 "주요 특성"보다 "설명용으로 선택한 네 개 특성"이라고 표현하는 것이 더 정확하다.

## Heart Disease 데이터셋

### 장점

- 남녀 모두에게 해당되는 질병이라 유방암보다 더 중립적으로 느껴질 수 있다.
- 이진 분류라 혼동 행렬, 정밀도, 재현율 설명에 좋다.
- `age`, `trestbps`, `chol`, `thalach`, `oldpeak` 같은 특성이 비교적 직관적이다.

### 단점

- scikit-learn 내장 데이터셋이 아니다.
- UCI 또는 Kaggle에서 별도로 받아야 한다.
- 범주형, 정수형, 실수형 특성이 섞여 있다.
- `sex` 변수가 포함되어 있으므로, 성별 변수를 쓰지 않으려면 제외해야 한다.

### 구하는 방법

- UCI Heart Disease Dataset:
  - https://archive.ics.uci.edu/dataset/45/heart+disease
- Python에서는 `ucimlrepo` 패키지로 받을 수 있다.

```python
from ucimlrepo import fetch_ucirepo

heart = fetch_ucirepo(id=45)
X = heart.data.features
y = heart.data.targets
```

### 보통 사용하는 13개 입력 특성

| 변수 | 의미 | 유형 |
| --- | --- | --- |
| `age` | 나이 | 수치형 |
| `sex` | 성별 | 범주형 |
| `cp` | 흉통 유형 | 범주형 |
| `trestbps` | 안정 시 혈압 | 수치형 |
| `chol` | 혈청 콜레스테롤 | 수치형 |
| `fbs` | 공복 혈당 > 120 mg/dl 여부 | 범주형 |
| `restecg` | 안정 시 심전도 결과 | 범주형 |
| `thalach` | 최대 심박수 | 수치형 |
| `exang` | 운동 유발 협심증 여부 | 범주형 |
| `oldpeak` | 운동으로 인한 ST depression | 수치형 |
| `slope` | 최대 운동 ST segment의 기울기 | 범주형 |
| `ca` | 형광투시로 확인된 주요 혈관 수 | 수치형/순서형 |
| `thal` | thalassemia 관련 검사 결과 | 범주형 |

타깃은 보통 `target` 또는 `num`으로 표현되며, 심장질환 여부를 나타낸다.

### 성별 변수를 제외한 예시 특성

처음에는 수치형 위주로 사용할 수 있다.

```python
numeric_features = [
    "age",
    "trestbps",
    "chol",
    "thalach",
    "oldpeak",
]
```

그다음 범주형 특성을 원-핫 인코딩해 추가할 수 있다.

```python
categorical_features = [
    "cp",
    "fbs",
    "restecg",
    "exang",
    "slope",
    "ca",
    "thal",
]
```

## 원-핫 인코딩 설명 방향

- 범주형 변수는 문자열이나 코드값으로 표현된 범주를 의미한다.
- 선형 회귀나 로지스틱 회귀 같은 모델은 입력값을 숫자로 받아 계산한다.
- 하지만 범주형 값을 단순히 `0`, `1`, `2`로 두면 모델이 순서나 크기 차이가 있다고 오해할 수 있다.
- 원-핫 인코딩은 각 범주를 별도의 0/1 열로 바꾸는 방법이다.
- 예를 들어 `island`가 `Biscoe`, `Dream`, `Torgersen` 세 값을 가진다면 다음과 같은 열로 바꿀 수 있다.
  - `island_Biscoe`
  - `island_Dream`
  - `island_Torgersen`
- 해당 범주이면 1, 아니면 0이 된다.
- 훈련셋과 테스트셋은 반드시 같은 원-핫 인코딩 열 구조를 가져야 한다.
